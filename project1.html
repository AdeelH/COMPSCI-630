1 Overview
For this assignment you will be replacing rusts default memory allocator with
a new custom built allocator. This assignment will give you insight into how
dynamic allocation of memory on the Heap functions.
2 Algorithm
You will implement an allocator that uses mmap from the nix rust library to
allocate a fairly large chunk of memory. You will then divide this memory
page into equally spaced buﬀers, each of which corresponds to a diﬀerent size
memory object. The ﬁrst bin will store object for 4 bytes, the second for 8
bytes and so on all the way up to 2048bytes, in powers of 2. You will then
allocate a stack for each buﬀer that keeps track of the freed objects. When
you go to allocate an object, ﬁrst check the stack for the respective bin. If
the stack is not empty, fetch the top item and return it. Otherwise, move
the pointer for the buﬀer forward by one and return the address. When you
go to free an item, just push onto the stack and return. The pseudo code for
this is as follows:
function init
memory← mmap(HEAP SIZE)
for all size classes do
store pointer to memory location
store start pointer for bin
1
store end pointer for bin
end for
end function
function alloc(size)
object class ← fetch object class ( size )
if object class is empty then
ptr ← address from object classes bump pointer
bump pointer move forward
else
ptr ←object class.pop()
end if
return ptr
end function
function free(ptr,size)
object class ← fetch object class ( size )
object class.push(ptr)
end function
3 What do you need to do?
We will provide you with three rust crates:
1. cheap
2. core printer
3. evaluate
cheap This is where you will implement your memory allocator. It follows
the memory allocator api described in Rust RFC 1974 and Rust RFC 1398.
You will need to implement the previously described algorithm by ﬁlling in
the alloc and dealloc methods. Alloc receives a Layout(deﬁned in the RFC’s)
that contains both the size of the requested object as well as its alignment.
Use this to request the appropriately sized block. Dealloc receives a pointer
as well as its size. You should use this to free the memory object.
2
core printer This crate is a helper crate that can be used to print strings,
pointers, and integers to the console. This will prove to be quite helpful
for debugging since you will have no access to the standard library in your
memory allocator. This crate’s functions can be called as follows:
1. print string(b”This will print\n”)
2. print int(902)
3. print u8(ptr)
evaluate This crate is a program that will call your allocator and make
several memory calls to test that your program functions correctly.
4 Evaluation
You will be evaluated on the following basis:
1. Correctness of Algorithm
2. Output of Correct Memory Statistics
3. Performance of Provided Benchmarks
4. Coding Style
In addition to the algorithm functioning correctly, your allocator will need
to print out several statistics on current memory usage. You should report
the current memory usage(size of all allocated blocks), available memory and
timing of memory operations.
